## 4. Риски и ограничения тестирования

| Риск | Вероятность | Влияние | Митигация |
|------|------------|---------|-----------|
| Медленные ответы `history.get` на большом окне (R-01) | Высокая | Высокое — тесты TC-042, TC-067 могут превышать таймаут | Ограничить `ZABBIX_PAGE_LIMIT`, добавить отдельный таймаут в интеграционных тестах; задокументировать ожидаемую задержку |
| Разный формат выражений триггеров в Zabbix 6.0 vs 7.0 (R-02) | Средняя | Среднее — TC-054 потребует двух вариантов выражений | Запускать TC-054 в двух конфигурациях CI-матрицы: против Zabbix 6.0 с legacy-форматом и против 7.0 с новым |
| Отсутствие реального Zabbix в среде CI | Средняя | Высокое — интеграционные тесты не выполнятся | Использовать Docker Compose (docker-compose.test.yml) для поднятия Zabbix 6.0/6.4/7.0; respx-моки покрывают юнит-тесты |
| Исчерпание тестовых данных при параллельном запуске тестов | Низкая | Среднее — гонки при создании/удалении одних и тех же объектов | Генерировать уникальные имена объектов с использованием UUID в фикстурах |
| Медленная сборка Docker-образов Zabbix в CI | Средняя | Низкое — увеличение времени CI | Кэшировать Docker-образы между запусками CI |
| bulk_update_macro на 500+ хостах создаёт нагрузку на Zabbix API (R-04) | Средняя | Среднее — тест TC-063 может замедлить прочие интеграционные тесты | Запускать TC-063 в отдельном job CI; изолировать тестовые данные |
